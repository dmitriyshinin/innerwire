# How to contribute / Как участвовать

## Principles / Принципы

InnerWire is an open project. We welcome contributions but ask that you follow a few principles.

InnerWire — открытый проект. Мы приветствуем участие, но просим соблюдать несколько принципов.

### 1. Scientific basis / Научная обоснованность
Every claim in the model must be backed by a reference to a peer-reviewed publication. Opinions, personal experience, and popular books are not sufficient grounds for inclusion (though they may be grounds for discussion).

Любое утверждение должно быть подкреплено ссылкой на рецензированную публикацию.

### 2. Honesty about evidence level / Честность об уровне доказательности
If research is preliminary — say so. If results are contradictory — show both sides. If something is a hypothesis — call it a hypothesis.

Если исследование предварительное — скажи об этом. Если результаты противоречивы — покажи обе стороны.

### 3. Accessible language / Доступность языка
Texts for the "main" level should be understandable without specialized education. Technical details go in collapsible sections (details/summary) or separate files.

Тексты для «основного» уровня должны быть понятны человеку без специального образования.

### 4. Bilingual / Двуязычность
Content should be provided in both Russian and English. If you can only contribute in one language, that's fine — note it in the PR and someone else can translate.

Содержание желательно предоставлять на обоих языках. Если можете только на одном — укажите в PR, кто-нибудь переведёт.

## What you can do / Что можно делать

- **Add sources** — found a relevant study? Suggest it in sources.md
- **Fix errors** — factual, grammatical, structural
- **Improve descriptions** — make them clearer, more accurate, more accessible
- **Suggest practices** — with mandatory scientific justification
- **Translate** — between Russian and English, or into other languages
- **Develop tools** — questionnaires, trackers, visualizations
- **Critique** — if you see a weak spot, open an issue

## PR format

1. One change — one PR
2. In the PR description: what changed and why
3. If a new claim is added — include a source reference
4. If an existing claim is changed — explain why the new formulation is more accurate
